# Load testing report

## 1. Настройка и запуск

### Требования

- Java 21+ (для поддержки виртуальных потоков)
- Установленная библиотека Gson (файл `gson-2.10.1.jar` в папке `lib`)

### Сборка и запуск

1. Открыть проект в корневой папке `3`.
2. В файле `src/PerformanceServer.java` настроить нужный режим:
   - флаг `USE_GSON` - использовать Gson (`true`) или собственную библиотеку (`false`)
   - флаг `USE_VIRTUAL_THREADS` - использовать виртуальные потоки (`true`) или классический пул (`false`)
   - При необходимости изменить количество потоков классического пула `THREAD_POOL_SIZE`
3. В командной строке перейти в корень проекта `3`:
   ```bash
   cd tasks\term-2\3
   ```
4. Компиляция сервера и тестового клиента:
   ```bash
   javac -cp "lib/*" src/PerformanceServer.java src/LoadTest.java
   ```
5. Запуск сервера (должен быть запущен до теста):
   ```bash
   java -cp "lib/*;src" PerformanceServer
   ```
   > Для изменения параметров сервера отредактируйте константы `USE_VIRTUAL_THREADS` `USE_GSON` `THREAD_POOL_SIZE` в `src/PerformanceServer.java`.
6. Во время работы сервера запустить нагрузочный тест:
   ```bash
   java -cp "lib/*;src" LoadTest
   ```
   > Для изменения параметров нагрузки отредактируйте константы `REQUESTS` и `CONCURRENCY` в `src/LoadTest.java`.


## 2. Описание эксперимента

Эксперимент измеряет среднее время обработки двух типов HTTP-запросов:

- **Request‑1**: сервер принимает JSON `{"data":"..."}`, парсит его, сохраняет строку в файл `storage.txt`, затем читает её обратно и возвращает JSON `{"stored":"...","retrieved":"..."}`.
- **Request‑2**: сервер принимает JSON `{"a":...,"b":...}`, парсит его, вычисляет сумму `a + b` в памяти и возвращает JSON `{"result":...}`.

Сравниваются четыре конфигурации:

- Собственный JSON парсер (из лабораторной №1) + виртуальные потоки
- Собственный JSON парсер + классический пул потоков
- Gson + виртуальные потоки
- Gson + классический пул потоков

## 3. Аппаратное обеспечение

- **Операционная система**: Windows 10
- **Процессор**: Intel Core i3
- **JVM**: Java 21
- **Оперативная память**: 8 ГБ
- **Дисковое пространство**: SSD

## 4. Параметры эксперимента

- **Количество параллельных клиентов (CONCURRENCY)**: 50
- **Общее количество запросов на каждый endpoint (REQUESTS)**: 100
- **Размер классического пула потоков сервера (THREAD_POOL_SIZE)**: 16
- **Тело запроса Request‑1**: `{"data":"test data"}` (размер ~25 байт)
- **Тело запроса Request‑2**: `{"a":123,"b":456}` (размер ~20 байт)
