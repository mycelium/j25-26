# Load testing report

Откройте командную строку и перейдите в корень пакетов — папку, внутри которой лежат папки src и lib.

## Компиляция

javac -cp "lib/*" src/*.java

## Запуск сервера

1. Классические потоки + собственный парсер

java -cp "lib/*" src/Main.java classic own

2. Классические потоки + gson

java -cp "lib/*" src/Main.java classic gson

3. Виртуальные потоки + собственный парсер
   
java -cp "lib/*" src/Main.java virtual own

5. Виртуальные потоки + gson
 
java -cp "lib/*" src/Main.java virtual gson

## Запуск нагрузочного теста

1. /store

java -cp "lib/*" src/Loadtest.java store

3. /compute

java -cp "lib/*" src/Loadtest.java compute

## Требования

Java 21+ (для виртуальных потоков).

## Experiment description

### Типы запросов
- Request‑1 (/store) – I/O нагрузка. Сервер принимает JSON {"id":123,"data":"..."}, парсит его (собственным парсером или Gson), записывает строку id:data в файл data.txt, затем считывает весь файл. Ответ: {"status":"ok","id":123}.

- Request‑2 (/compute) – CPU нагрузка. Сервер принимает JSON {"value":1000}, парсит, итеративно вычисляет факториал числа 1000 и возвращает {"result":...}.

### Комбинации
- Виртуальные потоки + собственный парсер

- Виртуальные потоки + Gson

- Классические потоки + собственный парсер

- Классические потоки + Gson

### Методика

- Параметры нагрузки: 50 параллельных потоков, 1000 запросов, 100 прогревочных запросов.

- Измеряется среднее время ответа (в миллисекундах), минимальное, максимальное и пропускная способность.

- Хост сервера -	localhost
  
- Порт сервера -	8080

- Параллельных потоков -	50

- Всего запросов -	1000

- Прогревочных запросов	- 100

- Размер JSON‑тела (Request‑1) -	44 байта

- Размер JSON‑тела (Request‑2) -	14 байт

- Файловая операция (Request‑1) -	запись одной строки + чтение всего файла

- Вычисление факториала (Request‑2) -	итеративное, n = 1000

Мини выводы: I/O нагрузка (/store) занимает значительно больше времени, чем CPU нагрузка (/compute). Виртуальные потоки существенно быстрее классических – это ожидаемо. Gson показал примерно ту же производительность, что и собственный парсер.
