# Load Testing Report 

## 1. Как настроить и запустить

### Параметры теста

Откройте `Main.java` и при необходимости измените константы:

```java
private static final int TOTAL_REQUESTS  = 500;  // кол-во запросов на каждый эндпоинт
private static final int CONCURRENT      = 50;   // параллельных клиентских потоков
private static final int WARMUP_REQUESTS = 50;   // прогревочных запросов перед замером
```

### Запуск

```powershell
# Удалить старую базу данных перед новым прогоном
Remove-Item data\storage.db -ErrorAction SilentlyContinue

# Собрать и запустить
.\gradlew.bat run
```

## 3. Описание эксперимента

Цель эксперимента — сравнить производительность HTTP-сервера при разных комбинациях двух параметров:

| Параметр | Вариант A | Вариант B |
|---|---|---|
| Тип потоков | Виртуальные  | Классические  |
| JSON-библиотека | Собственный парсер (лаб. 1) | Gson |

4 конфигурации тестируются последовательно. Каждая конфигурация запускает сервер заново, проводит прогрев JIT, затем измеряет два эндпоинта.

### Эндпоинты

**Request-1 — I/O нагрузка**  
`POST /request1`, тело: `{"value": "hello from load tester"}`

1. Парсит входящий JSON, извлекает поле `value`
2. Записывает строку в таблицу SQLite (`INSERT`)
3. Читает её обратно по сгенерированному `id` (`SELECT`)
4. Возвращает прочитанное значение

Каждый запрос совершает реальную запись и чтение из файла базы данных (SQLite в WAL-режиме, файл на диске).

**Request-2 — CPU нагрузка**  
`POST /request2`, тело: `{"n": 30}`

1. Парсит входящий JSON, извлекает поле `n`
2. Вычисляет число Фибоначчи F(30) итеративно
3. Сериализует результат в JSON
4. Возвращает `{"n": 30, "result": 832040}`

Всё вычисление происходит в памяти.

### Методика замеров

- Перед каждым замером выполняется **прогрев** (50 запросов) для разогрева JIT-компилятора
- Измеряется **среднее время ответа** на один запрос (мс), минимальное и максимальное
- Все запросы идут на `localhost` — сервер и клиент на одной машине
- Используется `java.net.http.HttpClient` (HTTP/1.1) без повторного использования соединений

| HTTP-версия клиента | HTTP/1.1 |
