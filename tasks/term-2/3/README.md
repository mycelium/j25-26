# Load testing report
## 1. Как настроить и запустить 
### Требования
- Java 21+ для поддержки виртуальных потоков,
- Установленная библиотека Gson
### Сборка и запуск
1. Открыть проект в VS Code.
2. В `ServerStart.java` настроить нужный режим:
    * флаг USE_GSON показывает использование библиотеки Gson
    * флаг IS_VIRTUAL показывает, используются виртуальные потоки
```bash
private static final boolean USE_GSON = false;
private static final boolean IS_VIRTUAL = false;
```
3. В командной строке перейти в корень проекта (пример: `cd tasks\term-2\3`)
4. Компиляция:
`javac -cp "libs/*;." -d out ServerStart.java`
5. Запуск сервера:
`java -cp "out;libs/*" ServerStart`
6. Во время работы сервера запустить `Main`:
    - через `Run Java` в VS Code
    - через новую командную строку:
        1. В командной строке перейти в корень проекта (пример: `cd tasks\term-2\3`)
        1. Компиляция `javac Main.java`
        2. Запуск `java Main`

## 2. Описание эксперимента
Эксперимент измеряет время, затраченное на выполнение операций, зависящих от диска (IO-bound) и от процессора (CPU-bound). В качестве IO-bound используется работа с файлом, для CPU-bound используются математические вычисления.

Сервер поддерживает 2 типа запросов:

* Request-1 (IO-bound): сервер принимает входящий POST-запрос с JSON, парсит его, сохраняет данные в текстовый файл, читает их из файла и возвращает ответ.
* Request-2 (CPU-bound): сервер принимает JSON, парсит его, выполняет вычисление количества простых чисел до (число + 1000), и возвращает результат в виде JSON. 

В процессе эксперимента сравниваются разные конфигурации, связанные с использованием потоков и парсеров:
* Собственный JSON парсер + виртуальные потоки
* Собственный JSON парсер + классический пул потоков
* Gson + виртуальные потоки
* Gson + классический пул потоков

* Дисковое пространство - 453 ГБ

## 4. Параметры эксперимента
* Количество потоков: 20
* Количество запросов на каждый поток: 100
* Запросов на каждый тест: 2000
* Потоков сервера: 4 (для классического пула) 
* Тело запроса и его размер: 	
  1. Request-1: ~30 байт, 
`{"id": 1, "value": "test"}`
  2. Request-2: ~20 байт, `{{"number": 42}}`
* Прогрев JVM: выполняется перед измерениями
* Измерения проводились на одной машине (сервер и клиент)
