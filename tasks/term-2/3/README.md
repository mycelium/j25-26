## Как настроить и запустить

### Требования
- Java 21+
- Apache JMeter 5.6.3

### Запуск сервера

1. Откройте проект в Eclipse
2. Откройте `runner/ServerRunner.java`
3. Установите параметры конфигурации:
```java
static final boolean IS_VIRTUAL     = false; // true = виртуальные потоки, false = классические
static final boolean USE_OWN_PARSER = true;  // true = мой парсер, false = Gson
```
4. Запустите `ServerRunner.java` 
5. Сервер запустится на `http://localhost:8080`

### Запуск JMeter

1. Откройте `C:\JMeter\bin\jmeter.bat`
2. Откройте файл `jmeter/load_test.jmx`
4. Нажмите **Start**
5. Результаты в **Summary Report** или **Aggregate Report**

### Запуск всех 4 конфигураций

Запускайте конфигурации по очереди, каждый раз перезапуская сервер:

| # | IS_VIRTUAL | USE_OWN_PARSER |
|---|---|---|
| 1 | false | true |
| 2 | false | false |
| 3 | true | true |
| 4 | true | false |

Перед каждым новым запуском необходимо очищать результаты JMeter.

## Описание эксперимента

Эксперимент измеряет производительность HTTP сервера под нагрузкой при разных комбинациях модели потоков и JSON парсера.

**Тестируются два эндпоинта:**

- `POST /store` — принимает JSON, парсит его, дописывает в файл `data/storage.json`, читает и возвращает последнюю запись
- `POST /calculate` — принимает JSON, парсит его, вычисляет сумму всех числовых полей, возвращает результат в виде JSON

**Сравниваются четыре конфигурации:**
- Виртуальные потоки + собственный JSON парсер
- Виртуальные потоки + Gson
- Классический пул потоков + собственный JSON парсер
- Классический пул потоков + Gson

JMeter и HTTP сервер запускались на одной машине.

## Параметры эксперимента

| Параметр | Значение |
|---|---|
| Количество потоков JMeter | 50 |
| Период разгона (Ramp-up) | 5 секунд |
| Запросов на поток | 100 |
| Всего запросов на один тест | 5000 |
| Размер тела запроса | ~40 байт |
| Тело запроса | `{"name":"Alice","age":25,"score":100}` |
| Потоков сервера (классический пул) | 10 |
| Файловое хранилище | `data/storage.json` (режим добавления) |
