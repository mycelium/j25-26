# Описание
В данной лабораторной работе  реализовано приложение на Java для анализа отзывов на фильмы и выставляет каждому оценку:
- positive
- negative
- neutral

## Используемые ресурсы
Датасет - [https://drive.google.com/file/d/15oxF9_ifxKMBs56eUIaziD4nRH3VUV9E](https://drive.google.com/file/d/15oxF9_ifxKMBs56eUIaziD4nRH3VUV9E)
Библиотека - [https://mvnrepository.com/artifact/edu.stanford.nlp/stanford-corenlp](https://mvnrepository.com/artifact/edu.stanford.nlp/stanford-corenlp)

Датасет содержит 50000 отзывов, из которых:

- 25000 — positive
- 25000 — negative

## Особенность CoreNLP и преобразование тональностей

В задаче используется несколько различных шкал тональности:

- Датасет IMDB содержит 2 класса  
  `positive`, `negative`

- Техническое задание требует 3 класса
  `positive`, `negative`, `neutral`

- Stanford CoreNLP возвращает 5 уровней тональности

| CoreNLP label | Score |
| ------------- | ----- |
| very negative | 0     |
| negative      | 1     |
| neutral       | 2     |
| positive      | 3     |
| very positive | 4     |

Таким образом необходимо выполнить преобразование 5-классовой шкалы CoreNLP → 3-классовую шкалу задачи.

### Алгоритм определения тональности

Для каждого отзыва выполняются следующие шаги:

1. Текст разбивается на предложения.
2. Для каждого предложения CoreNLP определяет уровень тональности.
3. Уровень переводится в числовую шкалу:

```
very negative -> 0
negative      -> 1
neutral       -> 2
positive      -> 3
very positive -> 4
```

4. Вычисляется средняя оценка по всем предложениям:

```
avgScore = sum(sentenceScores) / sentenceCount
```

### Преобразование в итоговый класс

Полученная средняя оценка переводится в требуемые 3 класса:

```
avgScore < 1.8  -> negative
avgScore > 2.2  -> positive
иначе           -> neutral
```

Такой подход позволяет учитывать разные тональности внутри одного отзыва и получать итоговую оценку для всего текста.

# Производительность  
  
Stanford CoreNLP является тяжёлой NLP-библиотекой.  
  
Основные причины медленной работы:  
  
1. используется синтаксический парсер  
2. каждый текст проходит полный NLP pipeline  
3. для анализа загружаются модели Stanford CoreNLP (100-150 мб)
  
Поэтому обработка большого количества отзывов занимает значительное время.  
  
Примерная скорость:  
  
| Reviews | Time            | 
| ------- | --------------- | 
| 5       | ~10–20 сек      | 
| 50      | ~1–2 минуты     | 
| 50000   | несколько часов |  
  
Поэтому в программе используется ограничение:
```java
private static final int MAX_REVIEWS_TO_ANALYZE = 5;
```
Это позволяет быстро проверить работу программы.

# Запуск программы
1) Положить файл и запустить без аргументов. 
```powershell
.\gradlew.bat run
```
2) Запустить с аргументами, где первый аргумент это путь до файла.
```powershell
.\gradlew.bat run --args="data/IMDB Dataset.csv"
```
