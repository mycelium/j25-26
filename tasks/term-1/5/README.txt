Реализовано приложение на Java, которое читает отзывы о фильмах из файла
и с помощью Stanford CoreNLP определяет, хороший отзыв, плохой или нейтральный.
Каждый отзыв — это одна строка в файле `reviews.txt` в папке ресурсов, пустые строки пропускаются.
При запуске программа для каждой строки считает тональность и выводит в консоль два столбца:
сначала метка `positive`, `negative` или `neutral`, потом сам текст отзыва;
ширина колонки для метки фиксированная, чтобы вывод выглядел ровно.

Внутри создаётся один общий объект пайплайна CoreNLP с нужными шагами обработки текста,
чтобы не загружать модель заново для каждого отзыва. Для анализа берутся все предложения в отзыве,
для каждого предложения считается своя тональность, а итоговый результат определяется так:
если больше позитивных предложений — отзыв считается позитивным, если больше негативных — негативным, во всех остальных случаях нейтральным.
Если модель не нашла ни одного предложения или текст пустой, отзыв тоже считается нейтральным
(по сути до этой проверки никогда не доходит, но это реализовано для безопасности, чтобы программа не падала в случае чего).
Класс тональности, который возвращает CoreNLP (строки вроде «Very positive» или «Negative»),
сначала приводится к удобному виду, а потом переводится в одну из трёх простых меток: положительный, отрицательный или нейтральный.

Проект собирается Gradle‑ом, используется Java 17 или новее, тесты написаны на JUnit 5.
Для выравнивания версий JUnit используется BOM‑зависимость org.junit:junit-bom:5.10.2
Для логирования подключён простой backend для SLF4J, чтобы не было предупреждения про то, что не найден класс `StaticLoggerBinder`, без него логирование по сути отключено.

Приложение запускается из корня проекта командой./gradlew run, после чего оно читает файл reviews.txt из ресурсов и печатает результат в консоль.
Отдельно есть тесты: один проверяет, что явно положительный отзыв даёт метку `positive`, другой — что явно негативный отзыв даёт `negative`,
третий проверяет, что «средний» по тону текст всё равно даёт одну из трёх допустимых меток и программа не падает, и ещё один тест проверяет, что пустой текст считается нейтральным.
Тесты запускаются через ./gradlew test
