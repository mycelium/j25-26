

### 7. Java: Parallel matrix multiplication
-   Реализовать функцию public static double[][] multiplyParallel(double[][] firstMatrix, double[][] secondMatrix);.
   
- Реализация должна использовать многопоточность.
-   Вывести время выполнения для матриц большой размерности.
 -  Сравнить время с однопоточной реализацией из лабораторной 1.
-   Подобрать оптимальное количество потоков под вашу платформу, описать подход к подбору в README.

### Выводы

- Ускорение после 8 потоков растёт не значительно, разница будет заметна только при больших матрицах (>=500)
- Оптимальным количеством потоков для моей платформы, при размере матрицы <= 1000, можно считать -  **8 потоков**, так как при 16 потоках выигрыш минимален при удвоенной нагрузке на систему.
- Параллельная реализация в среднем показывает ускорение до **≈10×** на больших матрицах по сравнению с однопоточной.
