# Результаты

Параллельная реализация умножения в среднем оказалась в 5.78 раз быстрее на 29 тредах для квадратной матрицы размера 1000.

# Треды

Количество тредов вычислялось следующим путем:
- создался цикл от 2 до 40 включительно (значения начали уменьшаться, поэтому большее количество итераций смысла не имело), где каждый цикл = количеству тредов в нем
- внутри цикла прогонялось 100 обычных и параллельных умножений, подсчитывалось среднее значение
- после было получено максимальное среднее из всех итераций
