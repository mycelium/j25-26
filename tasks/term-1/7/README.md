### Подбор оптимального количества потоков

Моя платформа с процессором Intel Core i7-4650U имеет 2 физических ядра, 4 логических потока. 

Количество потоков выбирается динамически:
- Для матриц размером 1500×1500 и больше используется фиксированное значение — 4 потока (максимум логических ядер).
- Для меньших матриц запускается автоматический подбор через метод `findOptimalThreadCount`.

Этот метод генерирует две случайные матрицы заданного размера, выполняет один «прогревочный» запуск для стабилизации работы JVM, а затем тестирует производительность при количестве потоков от 1 до 8 (ограниченном как min(2 × availableProcessors, 8)). Для каждого количества потоков проводится 5 замеров времени выполнения с использованием System.nanoTime(), после чего выбирается то количество потоков, при котором среднее время выполнения оказалось наименьшим.

Оптимальное количество потоков, определённое алгоритмом, в среднем составило 4.
