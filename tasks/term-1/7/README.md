# Parallel matrix multiplication
На матрицах размером меньше 150x150 параллельное вычисление занимает больше времени чем однопоточное. Так происходит, потому что создание потоков занимает много ресурсов, а ускорение от распараллеливания не настолько большое, чтобы нивелировать эти затраты. Далее представлены таблицы с результатами тестов для матриц размерами 500x500, 1000x1000, 2000x2000, для каждой матрицы было использовано разное количество поток, для определения оптимального количества.
### 500x500
| Количество потоков | Время выполнения (мс) |
|--------------------|-----------------------|
| 1                  | 189                   |
| 2                  | 120                   |
| 4                  | 77                    |
| 8                  | 61                    |
| 16                 | 64                    |
| 20                 | 67                    |
| 24                 | 79                    |
Для матрицы 500x500 самым оптимальным количеством потоков будет 8, хоть разница с 16 потоками совсем небольшая.

### 1000x1000
| Количество потоков  | Время выполнения (мс) |
|---------------------|-----------------------|
| 1                   | 3893                  |
| 2                   | 1867                  |
| 4                   | 1225                  |
| 8                   | 1264                  |
| 16                  | 1347                  |
| 20                  | 1329                  |
| 24                  | 1597                  |
Для матрицы 1000x1000 лучшим результатом было время при 4 потоках, время при 8 потоках отличается лишь немного, поэтому оптимальным количеством потоков можно считать 8. 

### 2000x2000
| Количество потоков  | Время выполнения (мс) |
|---------------------|-----------------------|
| 1                   | 57252                 |
| 2                   | 34124                 |
| 4                   | 23299                 |
| 8                   | 17910                 |
| 16                  | 17142                 |
| 20                  | 16927                 |
| 24                  | 16843                 |
Для матрицы 2000x2000 лучшее время было достигнуто при 24 потоках, но улучшение относительно 16 потоков минимальное, поэтому 16 потоков - это оптимальное значение.

### Выводы
Для небольших матриц <150x150 параллельное вычисление не приносит большой выгоды, а зачастую лишь увеличивает время вычислений. На матрицах среднего размера птимальное количество потоков 8. А для больших матриц от 2000x2000 выгоднее всего делить вычисления между 16 потоками.
