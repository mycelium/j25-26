# Java: Parallel matrix multiplication

Процессор имеет 12 логических потоков.

Для матриц размером меньше 120x120 лучше подходит однопоточная версия. Так как затраты
на создание потоков мешают получить хоть какое-то ускорение. А при неоптимальном выборе числа потоков
вообще замедляют работу более чем в 2 раза.

Для матрицы 500x500:

| Число потоков | Время выполнения |
|---------------|------------------|
| 1    		| 54 мс    	   |
| 2     	| 48 мс            | 
| 4   		| 26 мс            |
| 8  		| 14 мс            |
| 12  		| 15 мс            |
| 16  		| 14 мс            |

Для матрицы 500x500 оптимальным решением будет использовать 8 потоков. При увеличении числа потоком временные затраты или
не уменьшаются или уменьшаются незначительно.

Для матрицы 1000x1000:

| Число потоков | Время выполнения |
|---------------|------------------|
| 1    		| 870 мс    	   |
| 2     	| 552 мс           | 
| 4   		| 304 мс           |
| 8  		| 142 мс           |
| 12  		| 127 мс           |
| 16  		| 125 мс           |

Для матриц 1000x1000 лучший результат наблюдается при 12 потоках. Дальнейшее их увеличение не приводит
к значителььному улучшению.

Для матрицы 2000x2000:

| Число потоков | Время выполнения |
|---------------|------------------|
| 1    		| 6509 мс    	   |
| 2     	| 3962 мс          | 
| 4   		| 2153 мс          |
| 8  		| 1043 мс          |
| 12  		| 915 мс           |
| 16  		| 1046 мс          |

Для матриц 2000x2000 лучший результат также наблюдается при 12 потоках. При использовании большего числа потоков
вызываются дополнительные временные расходы, которые не дают получить выигрыш во времени.

## Вывод
Для совсем маленьких матриц лучше всего подходит однопоточная версия. Для матриц в интервале от 500x500 до 1000x1000 достаточно и 8 потоков
для достижения оптимального результата. Для матриц больше 1000x1000 лучший результат наблюдается при 12 потоках.
