# Описание
В данной лабораторной работе реализовано умножение матриц двумя способами:

- однопоточная версия `multiply(...)`
- многопоточная версия `multiplyParallel(...)`

Многопоточная реализация использует `ExecutorService` и пул потоков.  
Каждая задача вычисляет одну строку результирующей матрицы.

# Реализация

## Последовательная версия
В однопоточной реализации вычисление выполняется обычным тройным циклом:
```java
for (int i = 0; i < n; i++)
    for (int j = 0; j < b; j++) {
        double sum = 0;
        for (int k = 0; k < a; k++)
            sum += A[i][k] * B[k][j];
        C[i][j] = sum;
    }
```

## Параллельная версия
Основной метод параллельного умножения:
```java
public static double[][] multiplyParallel(double[][] A, double[][] B)
```
Этот метод является обёрткой и автоматически выбирает количество потоков, равное числу доступных процессорных ядер процессора:
```java
int threads = Runtime.getRuntime().availableProcessors();
```
После этого вызывается основной метод, который выполняет вычисления с заданным количеством потоков:
```java
multiplyParallel(A, B, threads);
```
где `threads` — количество потоков, используемых для вычислений.
### Основной алгоритм
В основной реализации создаётся пул потоков:
```java
ExecutorService pool = Executors.newFixedThreadPool(threads);
```
Далее для каждой строки результирующей матрицы создаётся отдельная задача.  
Каждая задача вычисляет одну строку результирующей матрицы.
```java
for (int i = 0; i < n; i++) {
    final int row = i;

    pool.execute(() -> {
        for (int j = 0; j < b; j++) {
            double sum = 0.0;

            for (int k = 0; k < a; k++)
                sum += A[row][k] * B[k][j];

            C[row][j] = sum;
        }
    });
}
```

После добавления всех задач пул потоков закрывается, и программа ожидает завершения выполнения всех потоков:
```java
pool.shutdown();
pool.awaitTermination(1, TimeUnit.HOURS);
```
# Подбор количества потоков

Для подбора оптимального количества потоков проводились экспериментальные измерения времени выполнения параллельного умножения матриц.

Алгоритм подбора:
1. задавался диапазон числа потоков от **2 до 8**
2. для каждого значения выполнялось **10 запусков**
3. вычислялось **среднее время выполнения**
4. оптимальным считалось количество потоков, при котором время было минимальным
# Измерение времени
Для тестирования использовались квадратные матрицы следующих размеров:
- 10 × 10
- 100 × 100
- 500 × 500
- 1000 × 1000
Для каждого размера:
- выполнялось **10 прогонов**
- вычислялось **среднее время выполнения**

# Анализ результатов

Для маленьких матриц (`10 × 10`, `100 × 100`) нету выигрыша от многопоточности.  
Это связано с тем, что само вычисление само по себе небольшое, а затраты на создание и управление потоками больше.
Для матриц большего размера (`500 × 500`, `1000 × 1000`) многопоточность даёт значительное ускорение.
Оптимальное количество потоков зависит от размера задачи:
- для `500 × 500` лучшим оказалось **7 потоков**
- для `1000 × 1000` лучшим оказалось **6 потоков**

Это связано с тем, что при большом числе потоков, они начинают конфликтовать за память компьютера.
