### 7. Java: Parallel matrix multiplication

- реализовать функцию `public static double[][] multiplyParallel(double[][] firstMatrix, double[][] secondMatrix);`
- Реализация должна использовать многопоточность
- Вывести время выполнения для матриц большой размерности
- Сравнить время с однопоточной реализацией из лабораторной 1
- Подобрать оптимальное количество потоков под вашу платформу, описать подход к подбору в README


### Подход к поиску оптимального количества потоков
Поиск оптимального количества потоков для параллельного умножения матриц выполняется автоматически при первом вызове метода multiplyParallel. Для этого создаётся тестовая пара матриц размером 1000×1000, и для каждого количества потоков от 1 до 2 × количество логических процессоров проводится 10 испытаний. Среднее время выполнения умножения для каждого количества потоков фиксируется, и выбирается то количество, при котором это время минимально.

На моем пк — процессор Intel Core i5-12400F (6 ядер, 6 потоков) — оптимальное количество потоков, определённое алгоритмом, в среднем составило 23. Это значение сохраняется и используется для всех последующих вызовов
