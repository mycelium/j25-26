# Лабораторная работа 7: Параллельное умножение матриц

## Задание
Реализовать параллельное умножение матриц с использованием многопоточности и сравнить с однопоточной версией.
## Результаты тестирования

### Матрица 500×500
Optimal number of threads: 5
Sequential: 85 ms
Parallel (5): 14 ms
Speedup: 6.07x

### Матрица 1000×1000  
Optimal number of threads: 14
Sequential: 696 ms
Parallel (14): 89 ms
Speedup: 7.82x


### Матрица 1500×1500
Optimal number of threads: 13
Sequential: 2379 ms
Parallel (13): 329 ms
Speedup: 7.23x

## Подбор оптимального количества потоков

### Методология:
1. **Автоматический поиск** в диапазоне 1-16 потоков
2. **Измерения**: 10 итераций + 2 прогрева JVM
3. **Критерий**: минимальное среднее время выполнения
4. **Speedup** = `время_однопоточное / время_параллельное`

### График производительности (1000×1000):

| Потоки | Время (мс) | Speedup |
|--------|------------|---------|
| 1      | 691        | 1.00x   |
| 4      | 227        | 3.04x   |
| 8      | 125        | 5.53x   |
| **14** | **89**     | **7.82x** |
| 16     | 102        | 6.77x   |

**Оптимальные значения:**
- 500×500: **5 потоков** (6.07x)
- 1000×1000: **14 потоков** (7.82x) 
- 1500×1500: **13 потоков** (7.23x)

### Анализ:

- 14 потоков оптимально для больших матриц (1000+)
- После 14 потоков - деградация (overhead > выигрыш)
- На малых матрицах (500) лучше меньше потоков
- Максимальный speedup ~7.8x при 16 ядрах

## Проверка корректности
Все результаты параллельной версии совпадают с однопоточной реализацией из первой лабораторной.
## Выводы
- **Оптимально 13-14 потоков** для моей 16-ядерной системы
- **Speedup до 7.8x** - отличный результат
- Параллелизация эффективна для матриц **≥1000×1000**
- Код стабильно находит минимум времени автоматически
