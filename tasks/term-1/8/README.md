##Запуск программы
Запуск осуществляется из корневой директории проекта
###1. Обучение модели и анализ ее точности
Если файл модели отсутствует, то программа начнет обучение (2 эпохи), если файл уже есть, то модель просто будет загружена для проверки и вывода ее статистик.
```bash
./gradlew run
```
Загрузится тестовый датасет MNIST, в консоль выведется статистика (Confusion Matrix и итоговая точность модели)
###2. Распознавание загруженного изображения
Можно передать путь к файлу (полный) в качестве аргумента командной строки:
```bash
./gradlew run --args="path/to/image.png"
```
Файл должен быть формата png или jpg с белой цифрой на черном фоне, в качестве примера в папке app находится файл two.png, при передаче этого файла в качестве аргумента был получен результат: "Result for file C:/JProjects/j25-26/tasks/term-1/8/app/two.png: 2".
Также был обновлен файл build.gradle.kts, были добавлены в dependencies все необходимые библиотеки.
