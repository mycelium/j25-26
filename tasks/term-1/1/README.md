В стандартном алгоритме умножения матриц используется тройной цикл в порядке i-j-k. Однако такой подход не всегда эффективен, так как данные матриц читаются из памяти не подряд, и процессор чаще делает кэш-промахи.

Чтобы ускорить выполнение программы, порядок циклов был изменён на i-k-j. При таком способе значение firstMatrix[i][k] используется сразу для всех столбцов j без повторного обращения к памяти. Это улучшает локальность данных и позволяет процессору эффективнее использовать кэш, что приводит к заметному ускорению вычислений на матрицах большого размера.
