# 6. Java: K-Nearest Neighbors (KNN) Classifier

Целью данной лабораторной работы является реализация алгоритма классификации K-ближайших соседей (K-Nearest Neighbors, KNN). Проект состоит из пяти основных классов, каждый из которых выполняет определенную функцию.

## Класс Point

Класс Point представляет собой основe проекта - точку в двумерном пространстве с координатами (x, y) и меткой класса.

Основные методы:
- Point(double x, double y, String label) - конструктор для тренировочных точек;
- Point(double x, double y) - конструктор для тестовых точек (без метки);
- distanceTo(Point other) - вычисляет евклидово расстояние до другой точки;
- Геттеры и сеттеры для координат и метки.

## Реализация алгоритма KNN

Класс KNNClassifier содержит основную логику алгоритма K-ближайших соседей.

Вспомогательный класс Neighbor:
```java
private static class Neighbor implements Comparable<Neighbor> {
    double distance;
    String label;
    
    @Override
    public int compareTo(Neighbor other) {
        return Double.compare(this.distance, other.distance);
    }
}
```
Класс Neighbor хранит пару "расстояние-метка" и реализует интерфейс Comparable для сортировки по расстоянию.

Основные методы:
- addTrainingPoint(Point point) - добавление одной тренировочной точки;
- addTrainingPoints(List<Point> points) - добавление коллекции точек;
- classify(Point testPoint) - основной метод классификации.

Особенности реализации метода classify:
- Используется евклидово расстояние для определения близости точек;
- Реализована защита от случая, когда k больше количества тренировочных точек;
- Добавлена валидация входных параметров.

## Визуализация данных

Класс DataVisualizer отвечает за создание графического представления точек и результатов классификации. Основные методы:
- visualizePoints() - основной метод создания изображения;
- drawGrid() - рисование координатной сетки;
- drawAxes() - рисование осей координат;
- drawLegend() - создание легенды классов;

## Основной класс приложения

Класс App содержит точку входа в программу (main метод) и демонстрирует работу алгоритма. Основные функции:
- Генерация тренировочных данных трех классов;
- Тестирование алгоритма на нескольких точках;
- Запуск классификации основной тестовой точки;
- Визуализация результатов.

Данные генерируются так: класс A - в левом верхнем квадранте (x: 1-4, y: 6-9), класс B - в правом нижнем квадранте (x: 6-9, y: 1-4), класс C - в центральной области (x: 3-7, y: 3-7).
